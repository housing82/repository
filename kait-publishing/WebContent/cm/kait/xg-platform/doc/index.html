Xg Platform API Documentation

개요

Xg Platform는 운영체제 및 브라우저에 추가적인 설치없이 동작할수 있도록 작성 되었으며 크게 데이터 통신 및 관리( Data Components ) 모듈과 사용자와 상호작용할 사용자 인터페이스( UI Components ) 모듈로 나누어 진다.

플랫폼과 컴퍼넌트는 개별사용이 가능하다. 단 분리사용 할 경우 컴퍼넌트의 데이터 바인딩과 같은 기능은 사용할 수 없다.

Category Prefix Description
Data Components xg- 서버( Gauce, InnoXync, Xena )와 통신 및 데이터 관리 방안 제공
UI Components xg-ui- 사용자와 상호작용하는 사용자 인터페이스 기능 제공

동작 구조도

브라우저 지원범위

Xg Platform은 하기 버전이상의 브라우저를 지원한다.

Chrome IE Fire Fox Opera Safari Android IOS
30+ 8+ 30+ 20+ 6+ 4.0+ 6.1+

폴더 구조

-+- lib      ( 의존성 모듈들이 포함되어있는 폴더 )
 +- doc      ( 현재 읽고 있는 API 문서 )
 +- examples ( 예제 폴더 )

설치

  1. Xg Platform 다운로드 링크를 통해 최신버전을 다운로드를 받는다.

  2. Xg Platform을 사용할 페이지에 스타일과 스크립트를 추가한다.

기본 방식

스크립트를 <head> 또는 <body> 태그안에 포함시키는 방식

{@lang xml}<head>
    ..( 생략 )..
    <link rel="stylesheet" type="text/css" media="all" href="xg-platform/lib/swidgets/styles/swidgets.css">

    <!-- 의존성 라이브러리 모듈 로드 -->
    <script src="xg-platform/lib/json3/lib/json3.min.js"></script>
    <script src="xg-platform/lib/jquery/dist/jquery.min.js"></script>
    <script src="xg-platform/lib/swidgets/swidgets.js"></script>

    <!-- xg-platform 모듈 로드 : 모듈을 개별로 로드해도 무관하다 -->
    <script src="xg-platform/xg-all.js"></script>
</head>

AMD 방식

require.js 사용하여 모듈을 비동기로 로드하는 방식

xg-platform 의존성 모듈( /lib ) 디렉토리에 require.js는 포함되어 있지 않다. 필요한 경우 개별 다운로드해서 사용하도록 한다.

{@lang xml}<head>
    ..( 생략 )..
    <link rel="stylesheet" type="text/css" media="all" href="xg-platform/lib/jswidgets/styles/swidgets.css">
</head>
<body>
    ..( 생략 )..
    <script data-main="js/main" src="lib/require.js"></script>
</body>

js/main.js

require.config( {
    paths: {
        'jquery': 'lib/jquery/dist/jquery.min',
        'json3': 'lib/json3/lib/json3.min',
        'swidgets': 'lib/swidgets/swidgets',
        'XgCommon': 'xg-common',
        'XgDataAdapter': 'xg-data-adapter',
        'XgDataTable': 'xg-data-table',
        ..( 생략 : 사용할 모듈 경로를 설정한다 )..
    }
} );

define( 'main', [ 
        'jquery', 
        'xgDataAdapter', 
        'XgDataTable' 
        ], function ( $, XgDataAdapter, XgDataTable ) {

            ..( 생략 )..
} );

위 스크립트 코드는 단순 예제이므로 사용할 모듈별로 설정은 유동적으로 변경한다.

사용방법

Xg Platform은 기본적으로 module 단위로 사용할 수 있으며, UI Components 경우에는 추가로 plugin, data-api 방식을 사용할 수 있다. 단 data-api 방식을 사용할 경우 주의사항이 있음으로 유의한다.

모듈 방식

new 키워드를 사용하여 인스턴스를 생성하는 방식

var dataTable = new XgDataTable();

var button = new XgUIButton( 'default', '.btn', { width: 100, height: 30 } );

플러그인 방식

jquery의 $.fn 등록되어 jquery 함수처럼 이용하는 방식

var $button = $( '.btn' );

$button.xgButton( { width: 100, height: 30 } );

$( '#btn-save' ).xgButton( { width: 100, height: 30 } );

data-api 방식

마크업에 data- 패턴의 속성을 정의하여 이용하는 방식

속성명은 data-xg- 패턴으로 시작하며 옵션을 적용하는 경우 data-xg-option이라는 속성에 JSON 문법으로 작성한다.

{@lang xml}<button type="button" 
        data-xg-button 
        data-xg-option='{ "width": 100, "height": 30 }'
>button</button>

data-api 방식 주의사항

  1. data-xg-option의 속성값은 JSON 문법으로 작성한다. JSON 문법규칙 상 문자열은 ""(쌍따옴표)로 묶여야 하고 모든 키값은 문자열로 작성되어야 한다. 따라서 data-xg-option 속성의 값은 반드시 ''(홀따옴표)로 묶여야 한다.

  2. 정적인 object type은 사용가능하나 참조형 데이터는 사용이 불가능하다.

  3. UI Components에 기능이 적용되는 시점은 브라우저에서 DOM 트리를 생성하고난 직후 최초 1회만 수행된다. 이후 DOM에 동적으로 data-xg-모듈명 패턴의 마크업이 삽입되더라도 기능이 적용되지 않는다.

JSON 문법은 여기를 참고한다.
