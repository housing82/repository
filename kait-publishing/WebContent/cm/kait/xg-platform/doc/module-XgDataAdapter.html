XgDataAdapter

서버와의 데이터 통신에 필요한 메서드를 제공한다.

Dependencies

Name Description Remark
XgCommon Xg Platform의 공통 상수 및 메서드를 제공한다.

CRUD(url, crud, dataTableIDs, packet, callback) → {object}

DataAdapter가 수행해야 할 동작을 실질적으로 담당하는 메쏘드이다. 생성/조회/갱신/삭제 등의 동작 수행을 서버에 요청한다.

Parameters:
Name Type Description
url string

별도로 지정된 데이터테이블 별 서버 주소
crud string

수행할 동작( C: 생성, R: 조회, U: 갱신, D: 삭제 )
dataTableIDs string

대상이 되는 DataTable ID의 String 배열
packet string

서버로 전송되어야 할 데이터
callback function

서버 응답이 오면 비동기적으로 호출되어야 할 callback 함수
Returns:

XgDataAdapter

Type
object
Example
da.CRUD(crud, dataTable, packet, function() {
     ...
});

getAsyncStatus() → {boolean}

현재 통신 시 동기 또는 비동기로 동작하는지 상태를 가져온다. false 이면 동기 방식으로 동작하는 것이며, true 이면 비동기 방식으로 동작하는 것이다.

Returns:

현재 비동기 설정 상태를 반환한다.

Type
boolean
Example
var async = da.getAsyncStatus();

Parameters:
Name Type Description

setSOCD(socd) → {object}

서버로 전달할 SOCD(Service Object Call Data) 문자열을 설정한다. DataTable을 INPUT/OUTPUT 용으로 구분하여 표기해 놓으며, 서버의 DataTable 이름과 클라이언트의 DataTable 이름을 매칭시켜준다. "I:EMPLOY1=DT1"는 '서버의 EMPLOY1 DataTable에 클라이언트 DT1 DataTable을 입력용으로 사용한다.'로 기술할 수 있겠다. 다수의 DataTable을 전송할 경우에는 콤마(,)를 이용하여 붙여 쓸 수 있다.

Parameters:
Name Type Description
socd string

서버로 전달할 SOCD(Service Object Call Data) 문자열
Returns:

XgDataAdapter

Type
object
Example
da.setSOCD("(O:dt_1=table01,O:dt_2=table02,I:dt_3=table03,I:dt_4=table04)");
