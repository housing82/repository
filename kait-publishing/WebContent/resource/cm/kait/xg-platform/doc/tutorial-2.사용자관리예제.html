2.사용자관리예제(30분)

본 문서는 Xg Platform을 시작하기 ( 5분 ) 내용을 실습해본 사용자들을 대상으로 작성되었다.

준비 사항

사용자의 하기의 폴더 구조 및 서버측에 API 가 준비되어 있다고 가정한다. 실제 서버가 없이 동작가능 하지만, 본 문서에는 서버와 연동을 한다는 가정하에 진행한다.

본 예제는 본인이 원하는 웹 서버 또는 웹 어플리케이션 서버 위에서 구동하도록 한다.

  • Xg Platform 다운로드 링크를 통해 최신버전을 다운로드를 받는다.
  • API URL
    • 사용자 정보 조회 : /XG/jsp/userSelect.jsp
    • 사용자 정보 변경 : /XG/jsp/userUpdate.jsp
    • 등급 정보 조회 : /XG/jsp/levelSelect.jsp
    --- style
     +- xg-platform +- doc
     |              +- examples
     |              +- lib
     index.html

작성할 어플리케이션

본 예제는 2개의 데이터 테이블을 이용한다. 하나는 사용자 정보를 관리하며, 나머지는 사용자 정보 중 등급의 대한 데이터를 제공할 역활을 수행한다.

사용자의 요구 사항은 다음과 같다.

  1. 그리드에는 일부분의 정보만 표현되고 모든 정보는 다른 영역( 입력 영역 )에 표현되어야 한다.
  2. 그리드 및 입력 영역의 내용을 수정할 수 있어야 한다.
  3. 그리드에서 선택된 로우가 변경될 때 마다 입력 영역의 내용이 변경되어야 한다.
  4. 그리드에 필터링, 그룹핑, 페이징, 정렬 기능이 제공되어야 한다.
  5. 입력 내용 중 키 및 시퀀스에 해당하는 컬럼은 신규입력의 경우에만 입력이 가능해야 하고, 서버에 반영된 이후에는 수정이 불가능해야 한다.
  6. 특정 컬럼의 경우 지정된 값중에 선택 가능한( 콤보박스, 체크박스, 드랍다운 리스트 등 ) 컴퍼넌트 형식으로 제공되어야 한다.

4. 스크립트 코드 작성 ( main.js )

// dom이 로드 된 후 수행한다.
$( document ).ready( function () {

    var dataSet = new XgDataSet();

    // 사용자 정보를 관리할 데이터 테이블
    var userDataTable = new XgDataTable( 'userDataTable', {
        url: {
            select: '/XG/jsp/userSelect.jsp',
            update: '/XG/jsp/userUpdate.jsp'
        },

        booleanColumn: [
            {
                column: 'MARRIAGE',
                trueValue: 'Yes',
                falseValue: 'No'
            }
        ]
    } );

    // 사용자 등급을 관리할 데이터 테이블
    var levelDataTable = new XgDataTable( 'levelDataTable', {
        url: { 
            select: '/XG/jsp/levelSelect.jsp' 
        }
    } );

    // 각각의 데이터 테이블을 데이터 셋에 등록
    dataSet.addDataTable( userDataTable );
    dataSet.addDataTable( levelDataTable );

    // 데이터 어댑터 생성 및 데이터 셋에 등록
    dataAdapter = new XgDataAdapter( 'dataAdapter' );
    dataAdapter.setDataType( 'XML' );
    dataAdapter.setSOCD( '(O:user=userDataTable,I:user=userDataTable,O:level=levelDataTable)' );

    dataSet.setDataAdapter( dataAdapter );

} );

dom을 접근하는 코드가 들어가기 때문에 dom이 로드 된 후 수행될 콜백 함수에서 코드를 작성한다. 'userDataTable' 생성 옵션에 추가된 'booleanColumn' 옵션은 'MARRIAGE' 컬럼값( 'Yes', 'No' )을 boolean 형태로 변경하여 checkbox에 사용하기 위함이다. 값이 변경되어 서버에 변경사항 전달 시 해당 boolean값이 다시 기존 값으로 변경되어 전달된다. 

    ..( 생략 )..

    // 각 input 별로 동일한 옵션을 사용하기에 공통 옵션을 생성하는 함수 등록
    var createOption = function ( columnName, additionalOption ) {

        var mergedOption = new Object();

        var option = {
            xgInitOption: {

                    column: columnName
                }
            },
            width: 180,

        }

        $.extend( true, mergedOption, option, additionalOption );

        return mergedOption;
    };

        template: 'info',

        dataSet.select( 'userDataTable' );

    new XgUIButton( 'default', '.update', {

        width: 150,
        height: 40
    } ).on( 'click', function () {

        $grid.xgGrid( 'addrow', null, {}, 'top' );
        $grid.xgGrid( { selectedrowindex: 0 } );

    } );

데이터 어댑터 등록 이후 input 계열 컴퍼넌트에서 공통으로 사용할 옵션을 생성해주는 함수를 정의한다. 이는 본 예제에서 사용하는 옵션이 대부분 비슷하여 재사용성을 높이기 위함이다. 해당 함수는 내부에서 사용하는 'option' 오브젝트와 'additionalOption' 오브젝트를 병합하여 'mergedOption' 생성 후 반환한다.

이후 그리드 변수선언 및 조회, 수정, 추가버튼을 모듈 형식으로 초기화( 플러그인 방식으로 사용해도 무관하다 ) 하고 필요에 따른 이벤트 리스너를 등록한다.

그리드 컴퍼넌트 변수선언을 이곳에서 먼저하는 이유는, 이후 작성되는 코드의 콜백 함수에서 해당 변수로 접근해야 함으로 먼저 선언하였다.

  • 조회 버튼 : 클릭이 일어나면 dataSet에 'userDataTable'의 데이터를 다시 받아온다.

  • 수정 버튼 : 클릭이 일어나면 dataSet에 'userDataTable'의 변경사항( 추가, 수정, 삭제 )을 서버에 전달한다. dataSet의 'update'의 함수는 추가 콜백이 전달되지 않으면 자동으로 'select' 함수를 호출한다.

  • 추가 버튼 : 클릭이 일어나면 그리드 첫번째 로우에 빈 로우를 추가하고, 해당 로우를 선택한다.( 이렇게 함으로 써 사용자가 상세정보의 입력컨트롤에 바로 추가된 로우에 내용을 입력 가능할 것이다 )

이후 데이터 테이블을 로드한다. 'levelDataTable'을 먼저 로드 후 이후 콜백에서 'userDataTable'을 로드 하는 이유는, 'userDataTable' 로드 이후 수행 할 콜백함수에서 그리드를 초기화하면서 등급에 대한 정보를 사용해야하는데 해당 내용은 'levelDataTable'이 로드가 되어야 참조 할 수 있음으로 먼저 로드를 하는 것이다.

'userDataTable'이 로드 된 후 해당 정보를 기준으로 그리드의 초기화를 진행한다. 그리드는 하기의 옵션으로 생성할 것이다. 이 외의 다양햔 옵션은 여기를 참조한다.

    • 그리드 셀 편집 모드에서 컬럼 사이즈 및 제약조건 값( 서버 응답의 헤더 기준 )을 기준으로 유효성 검사
    • 컬럼이 숫자형( 정수, 소수 )일 경우 기본 데이터 표시 포멧 지정( eg. 3자리 ',' 구분, 소수는 기본 2자리 까지 출력 )
    • 로우의 상태 컬럼 표시
    • 컬럼의 타입에 따라( 서버 응답의 헤더 기준 ) 컬럼 에디터 자동 적용( numberinput, datetimeinput )
  • 바인딩 템플릿 옵션중 'useEditor' 커스텀 옵션 기능 : 활성화
    • LEVEL 컬럼 : 드랍다운 리스트로 적용 및 'levelDataTable'내용을 기준으로 참조 데이터 정의
    • GENDER 컬럼 : 드랍다운 리스트로 적용 및 참조 데이터를 정적값으로 정의
    • REG_DATE 컬럼 : 날짜입력컨트롤로 적용
    • MARRIAGE 컬럼 : 체크박스로 적용 및 모두 체크 기능 활성화
  • 그룹 시 'AGE' 정보를 기준으로 소계식 적용 : 활성화 [ 'sum', 'avg', 'min', 'max', 'count' ]
