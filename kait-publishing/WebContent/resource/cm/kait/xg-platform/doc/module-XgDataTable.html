addRow(rowData, firstRowYn) → {object}

로우 데이터를 추가한다. 추가된 로우는 데이터 테이블의 마지막 로우에 들어간다. rowData가 생략 되었을 경우 빈 배열이 추가된다. 배열의 크기는 데이터 테이블 헤더의 총 컬럼 수와 일치하여야 한다.

Parameters:
Name Type Argument Description
rowData array

로우 데이터 배열 또는 로우 데이터 배열이 들어있는 배열( multiple )
firstRowYn boolean <optional>

firstRow를 통한 추가인지 여부
Returns:

xgDataTable

Type
object
Example
// 단일 로우 추가
xgDataTable.addRow([25, 'foo']);

// 다중 로우 추가
xgDataTable.addRow([[15, 'bar'], [41, 'hoo']]);

find(colIdx, value, startRowIdx) → {number}

xgDataTable 특정 컬럼( colIdx )에서 value와 일치하는 최초 로우의 인덱스를 반환한다. 일치하는 row가 없을 경우 -1을 반환한다. value 인자로 배열이 전달된 경우 해당 배열요소 중에 컬럼의 값이 포함되어 있다면 해당 로우의 인덱스 값을 반환한다.

Parameters:
Name Type Argument Description
colIdx number

컬럼의 인덱스
value unknown

찾을 값
startRowIdx number <optional>

시작 로우의 인덱스
Returns:

rowIdx value값이 같은 컬럼의 로우 인덱스

Type
number
Example
var findAge = xgDataTable.find(0, 30, 0);
var findUserId = xgDataTable.find(1, ['foo', 'bar'], 1);

setBooleanColumn(arr) → {object}

Server측에서 boolean 형식의 데이터를 보내줄 수 없어 boolean 형태로 표현되어야 하는 컬럼의 정보를 설정한다. 이후 서버에서 데이터 수신 시 해당 정보를 기준으로 boolean 형태로 설정된다. 데이터를 설정 시 trueValue와 같지 않은 값은 모두 false로 설정된다. 단 현재 표시되는 정보일 뿐이지 데이터가 변경되지 않으면 지정한 falseValue 로 저장되지 않는다.

해당 메서드는 서버 데이터가 동기화 되기 전에 설정하여야 반영된다.

Parameters:
Name Type Description
arr array

boolean 형식으로 표현 할 컬럼 정보 배열
Returns:

xgDataTable

Type
object
Example
var booleanColumn = [
    {
        column: 'age',
        trueValue: 30,
        falseValue: 40
    }
];

xgDataTable.setBooleanColumn(booleanColumn);

useOverrideColumn(bool) → {object|boolean}

true로 설정하면 클라이언트 측에서 정의한 컬럼정보를 기준으로 사용한다. false로 설정시 클라이언트에서 컬럼정보를 설정하더라도 서버응답의 헤더정보를 기준으로 사용된다. 설정후에는 자기 자신을 반환한다.

인자를 생략할 경우 현재 설정된 overrideColumn 값을 반환한다.
